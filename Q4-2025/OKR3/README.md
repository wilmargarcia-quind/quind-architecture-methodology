# OKR3: Diseñar y lanzar un nuevo servicio especializado en arquitectura de soluciones de software

Este repositorio contiene el avance del OKR3 del Q4.

## Key Results
- [ ] **KR1**: [KR1-Alcance-Funcional-Tecnico](./KR1-Alcance-Funcional-Tecnico/README.md)
  - Definir el alcance funcional y técnico del servicio, incluyendo tipos de arquitectura (monolítica, microservicios, orientada a eventos, serverless) y sus casos de uso aplicables.
- [ ] **KR2**: [KR2-Paquetes-Servicio](./KR2-Paquetes-Servicio/README.md)
  - Documentar al menos 3 paquetes de servicio con enfoque en escenarios reales (modernización, escalabilidad, integración).
- [ ] **KR3**: [KR3-Guia-Metodologica](./KR3-Guia-Metodologica/README.md)
  - Desarrollar una guía metodológica completa para la prestación del servicio, incluyendo roles, entregables y marco tecnológico.
- [ ] **KR4**: [KR4-Kit-Consultoria](./KR4-Kit-Consultoria/README.md)
  - Desarrollar un kit de consultoría en arquitectura de soluciones que incluya al menos 5 elementos clave (guía de diagnóstico, plantilla de entregables, checklist técnico y casos de uso).
- [ ] **KR5**: [KR5-Material-Comercial](./KR5-Material-Comercial/README.md)
  - Elaborar y comunicar 3 materiales clave para el equipo comercial (pitch, propuesta de valor, presentación de casos de uso) que permitan ofrecer el servicio a clientes potenciales.
